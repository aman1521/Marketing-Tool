# 🌐 Pulse System (Gravity Layer)

The macro-awareness layer of the Marketing Intelligence OS.

Pulse evaluates factors entirely outside the scope of individual creatives (Hawkeye) or specific account optimizations (CaptainDiagnose). It evaluates **Platform Algorithms, Consumer Demand Shifts, Auction Volatility, and Date Seasonality** to emit *Gravity Modifiers*.

## 🏗 Architecture Components

1. **`SeasonalityEngine`**: Predicts inherent cycles identifying Black Friday peaks or Summer lulls, shifting mathematical velocity equations natively.
2. **`DemandEngine`**: Recognizes structural collapse across global conversion arrays, indicating the issue isn't a bad ad (Fatigue) but a global recession/outdated product loop structurally.
3. **`VolatilityEngine`**: Computes standard deviations on platform auction prices. If CPMs multiply by 800% overnight seamlessly, it trips chaos flags.
4. **`MacroDriftEngine`**: Takes anonymized composite data representing system-wide API efficiency evaluating Meta or Google delivery architectures for algorithm changes.
5. **`PulseEngine`**: Merges the Gravity into a single `Confidence Modifier` scalar value structurally reducing scaling aggressiveness without blocking the logic layer manually.

## 🛡 Validations & Safety

- **Strict Dependencies**: Pulse *Never* Executes actions. It sits entirely as a scalar multiplier modifying `CaptainDiagnose_Confidence * Pulse_Confidence` guaranteeing safe failure thresholds.
